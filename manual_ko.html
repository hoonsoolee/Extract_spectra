초분광 노지 작물 분석 도구

비지도학습 전용 — 라벨 없이 사용 가능

이 도구는 초분광 노지 작물 이미지의 픽셀을 자동으로 분류하고 클래스별 반사율 스펙트럼을 추출합니다. 인터넷·클라우드·라벨 데이터 없이 내 컴퓨터에서 완전히 실행됩니다. 결과는 CSV 파일과 인터랙티브 HTML 리포트로 저장됩니다.

🌱 Hybrid (NDVI + K-means)
📊 K-Means 클러스터링
📐 스펙트럼 각도 매핑 (SAM)
🤖 Autoencoder (딥러닝)
📄 인터랙티브 HTML 리포트
📈 클래스별 스펙트럼 CSV
GitHub: https://github.com/hoonsoolee/Extract_spectra   |   Python 3.9 이상  |  CPU 전용 실행  |  Windows / macOS / Linux

설치 순서

  1. 터미널(명령 프롬프트)을 엽니다.
    Windows: Win+R → cmd 입력 후 엔터

  2. GitHub에서 코드를 내려받습니다:

    git clone https://github.com/hoonsoolee/Extract_spectra.git
cd Extract_spectra

  3. 필요한 파이썬 패키지를 한 번에 설치합니다:

    pip install -r requirements.txt

  4. (선택) PyTorch 설치 — Autoencoder 방법 사용 시에만:

    pip install torch

  5. 앱을 실행해 설치를 확인합니다:

    python -m streamlit run app_unsupervised.py

    브라우저가 자동으로 http://localhost:8501에서 열리면 성공입니다.

기본 추천

Hybrid (NDVI + 밝기 + K-means)

식생 물리량을 먼저 활용한 후 통계 클러스터링. NIR 밴드가 있는 일반 작물 이미지에 최적.

범용

K-Means

순수 데이터 기반 클러스터링. 스펙트럼 범위 무관하게 동작. 식생 지수 불필요.

조명 불변

SAM — 스펙트럼 각도 매핑

스펙트럼의 밝기가 아닌 모양(각도)을 비교. 그림자가 많거나 조명이 불균일한 이미지에 적합.

딥러닝

Autoencoder

신경망으로 스펙트럼을 압축한 후 클러스터링. PCA가 놓치는 비선형 패턴 포착. PyTorch 필요.

5.1 Hybrid — 기본 추천 방법

세 단계 기법을 순서대로 결합합니다. 식생 물리학을 활용하므로 순수 통계 방법보다 작물 이미지에서 직관적인 결과를 냅니다.

NDVI = (NIR − Red) / (NIR + Red)
임계값 이상 → 식생
미만 → 비식생
→
2
밝기 마스크
식생 픽셀 중:
평균 반사율 낮음 → 그림자 잎
높음 → 햇빛 잎

3
K-Means 세분화
각 세그먼트 내에서 K-means로 세부 클래스 생성 (품종 차이, 토양 유형 등)

파라미터

파라미터기본값범위역할
NDVI 임계값 0.15 0.05 – 0.30 이 값 이상인 픽셀을 식생(잎)으로 분류합니다.
↑ 높이면: 비식생이 잎 클래스에 섞이는 경우.
↓ 낮추면: 잎 픽셀이 너무 적게 검출되는 경우 (스트레스·노화 작물).
밝기 임계값 0.08 0.03 – 0.20 식생 픽셀 중 평균 반사율이 이 값보다 낮으면 그림자 잎으로 분류합니다.
↑ 높이면: 그림자가 많은 이미지에서.
↓ 낮추면: 밝은 환경이나 흐린 날 촬영된 이미지에서.

출력 클래스 ID

ID클래스 이름설명
0배경 (Background)식생도 토양도 아닌 픽셀 (경로, 가장자리 등)
1햇빛 잎 (Sunlit Leaves)직사광선을 받는 잎 — 높은 반사율
2그림자 잎 (Shadowed Leaves)그늘진 잎 — 낮은 반사율
3토양 (Soil)나지, 지표면
4기타 (Other)미분류 픽셀
💡
파라미터 조정 순서 기본값으로 한 번 실행합니다. HTML 리포트의 클래스별 이미지를 확인합니다. 한 클래스에 서로 다른 재질이 섞여 있으면 관련 임계값을 조금씩 바꿔가며 재실행합니다. 보통 1~2번 조정으로 충분합니다.
⚠
NIR 밴드가 없는 경우 NDVI 계산에는 근적외선(~800 nm)과 적색(~660 nm) 밴드가 필요합니다. 해당 밴드가 없으면 Hybrid 방법이 자동으로 K-Means로 전환됩니다. 이 경우 처음부터 K-Means 또는 SAM을 선택하는 것을 권장합니다.

5.2 K-Means

고전적인 비지도 클러스터링 알고리즘입니다. 식생 지수나 특정 밴드에 대한 가정 없이 순수하게 데이터의 통계적 구조만 사용합니다.

동작 원리

  1. PCA(주성분 분석)로 수백 개의 스펙트럼 밴드를 15개 주성분으로 압축합니다. 대부분의 분산을 유지하면서 노이즈를 제거합니다.

  2. K-Means가 압축된 스펙트럼 공간을 K개 클러스터로 분할합니다. 각 픽셀을 가장 가까운 클러스터 중심에 반복적으로 할당합니다.

  3. 각 클러스터가 하나의 출력 클래스가 됩니다. 클래스에는 색상이 지정되지만 이름은 없습니다 — 리포트의 스펙트럼 차트를 보고 직접 해석하세요.

파라미터

파라미터기본값역할
클래스 수 6 찾을 클러스터 수입니다. 클러스터가 많을수록 세밀하게 분류되지만 해석이 어려워집니다. 6으로 시작하고 리포트 확인 후 조정하세요.
💡
K-Means를 사용할 때 NIR 밴드가 없어 NDVI를 계산할 수 없을 때, 식생 지수 가정 없이 순수 데이터 기반 결과를 원할 때, 또는 Hybrid 결과와 비교 기준이 필요할 때 사용하세요.

5.3 SAM — 스펙트럼 각도 매핑

SAM은 스펙트럼 벡터 사이의 각도를 기준으로 유사도를 측정합니다 — 유클리드 거리(밝기 차이)가 아닙니다. 따라서 조명 강도 변화에 영향을 받지 않습니다. 같은 재질의 햇빛 픽셀과 그림자 픽셀은 반사율 값이 달라도 각도는 작습니다.

동작 원리

  1. Endmember 추출: PCA를 이용해 이미지에서 대표 스펙트럼을 자동으로 추출합니다. 별도의 참조 스펙트럼이 필요 없습니다.

  2. 각도 비교: 각 픽셀 스펙트럼과 모든 Endmember 간의 각도(라디안)를 계산합니다. 각도가 작을수록 유사합니다.

  3. 클래스 할당: 가장 작은 각도의 Endmember 클래스에 픽셀을 할당합니다.

  4. 배경 임계값 처리: 최솟값 각도가 임계값을 초과하면 해당 픽셀을 배경(클래스 0)으로 할당합니다 — 어떤 클래스와도 충분히 유사하지 않은 픽셀입니다.

파라미터

파라미터기본값역할
각도 임계값 0.10 rad (약 5.7°) 픽셀과 가장 가까운 Endmember 사이의 최대 허용 각도입니다. 초과 시 배경 처리됩니다.
↑ 높이면: 배경 픽셀이 너무 많을 때 (임계값이 너무 엄격).
↓ 낮추면: 더 엄격하게 매칭.
클래스 수 6 추출할 Endmember(클래스 대표 스펙트럼) 수입니다.
💡
SAM을 사용할 때 일출/일몰 직후 촬영, 부분 구름 조건, 지형 그림자 등 조명이 불균일하거나 그림자 편차가 큰 이미지에 특히 유용합니다.

5.4 Autoencoder (딥러닝)

작은 신경망으로 각 픽셀의 스펙트럼을 압축된 표현으로 학습한 후, 그 공간에서 K-Means 클러스터링을 수행합니다. PCA(K-Means 방법에서 사용)가 놓칠 수 있는 비선형 스펙트럼 패턴을 포착할 수 있습니다.

동작 원리

  1. 인코더: MLP가 수백 밴드의 스펙트럼을 16차원 잠재 공간으로 압축합니다.

  2. 디코더: 다른 MLP가 16차원 코드에서 원본 스펙트럼을 재구성합니다. 재구성 오차를 최소화하는 방향으로 학습됩니다.

  3. 클러스터링: 학습 완료 후 16차원 잠재 벡터에 K-Means를 적용해 최종 클래스를 결정합니다.

파라미터

파라미터기본값역할
학습 Epochs 60 학습 데이터 전체를 반복하는 횟수입니다. 많을수록 압축 품질이 높아지지만 시간이 더 걸립니다. 빠른 테스트에는 20~30, 복잡한 이미지에는 100~150이 적합합니다.
클래스 수 6 잠재 공간에 적용하는 K-Means 클러스터 수입니다.
⚠
PyTorch 필요 Autoencoder 방법은 PyTorch가 설치되어 있어야 합니다:
pip install torch
설치되어 있지 않으면 분석 시작 시 오류가 발생합니다.

HTML 리포트 보는 법

결과 리포트의 각 섹션 상세 설명

실행 후 출력 폴더에 report_YYYYMMDD_HHMMSS.html 파일이 저장됩니다. 모든 브라우저(Chrome, Firefox, Edge, Safari)에서 열 수 있습니다. 완전히 독립 실행형 파일이므로 인터넷이 필요 없습니다.

9.1 이미지 개요

첫 번째 카드에 세 가지 이미지가 나란히 표시됩니다:

이미지사용 밴드확인 포인트
RGB 합성 Red ~660 nm, Green ~550 nm, Blue ~450 nm 자연색 화면. 데이터가 제대로 로드됐는지, 장면이 예상대로인지 확인합니다.
CIR 위색도 NIR ~800 nm → Red, Red → Green, Green → Blue 건강한 식생이 선명한 빨강/분홍으로 보입니다. 분류 맵의 잎 클래스 위치와 일치하는지 시각적으로 검증합니다.
통합 분류 맵 전 밴드 (알고리즘 출력) 모든 클래스를 색상으로 표시하고 범례에 픽셀 수와 비율을 보여줍니다. 전체 분류 결과를 한눈에 확인합니다.

9.2 클래스별 분류 이미지

클래스마다 큰 이미지 하나씩 표시됩니다. 해당 클래스에 속하는 픽셀만 클래스 색상으로 강조하고 나머지는 어두운 흑백으로 표시합니다. 각 클래스를 독립적으로 육안 검사하기에 최적입니다.

좋은 분류 결과:

클래스 픽셀이 공간적으로 일관된 영역을 형성합니다 — 예: 전체 엽관 영역, 연속된 토양 구간, 이랑의 그림자 면.

조정이 필요한 경우:

클래스 픽셀이 무작위로 흩어져 있거나, 서로 다른 재질(예: 토양과 잎)이 한 클래스에 섞여 있습니다. 파라미터를 조정 후 재실행하세요.

9.3 분류 요약 표

각 클래스의 픽셀 수, 전체 이미지 면적 대비 비율, 비율을 시각화한 색상 막대가 표시됩니다.

분포가 장면에 맞는지 확인하세요. 밀도 높은 작물 이미지라면 잎 클래스가 크게, 드문드문 심어진 작물이라면 토양/배경 클래스가 크게 나와야 자연스럽습니다.

9.4 반사율 스펙트럼 차트

각 클래스의 평균 반사율 스펙트럼을 파장(nm) 또는 밴드 인덱스에 따라 그린 인터랙티브 Plotly 차트입니다. 선 주변의 음영 영역은 ±1 표준편차로 클래스 내 스펙트럼 변동성을 나타냅니다.

조작방법
특정 파장 값 확인마우스를 차트 위에 올리면 팝업 표시
특정 클래스 표시/숨기기범례에서 해당 클래스 이름 클릭
하나의 클래스만 보기범례에서 해당 클래스 이름 더블클릭
확대차트에서 드래그
확대 초기화차트에서 더블클릭
PNG로 저장차트 위에 마우스 올리기 → 우상단 카메라 아이콘 클릭
💡
스펙트럼 해석 포인트 클래스 간 스펙트럼 선이 대부분의 파장대에서 수직으로 명확히 분리될수록 좋은 분류입니다. 두 클래스의 스펙트럼이 거의 겹친다면 클래스 수를 줄이는 것을 고려하세요.

9.5 품질 평가

클러스터가 스펙트럼 공간에서 얼마나 잘 분리됐는지 두 가지 지표로 자동 계산합니다.

클러스터 품질 점수 (Silhouette 기반 게이지)

Silhouette Score(범위: −1 ~ +1)는 각 픽셀 스펙트럼이 자신의 클래스와 얼마나 가깝고 다른 클래스와 얼마나 먼지를 측정합니다. 가독성을 위해 0~100%로 변환됩니다.

0% — 불량 25% 50% — 보통 75% — 양호 100% — 우수
점수등급의미
90 – 100%우수클래스가 스펙트럼 공간에서 매우 잘 분리됨.
75 – 90%양호대부분의 과학적 목적에 충분한 신뢰도.
50 – 75%보통클래스 간 겹침 있음. 파라미터 조정 또는 다른 방법 시도 권장.
50% 미만불량심각한 겹침. 다른 방법 선택, 클래스 수 변경, 임계값 조정 필요.

Davies-Bouldin 지수

클러스터 내 분산과 클러스터 간 거리의 비율을 측정합니다. 낮을수록 좋습니다. 1.0 이하는 일반적으로 양호, 2.0 이상은 클래스 간 겹침이 심함을 시사합니다. 절대적 기준보다는 같은 이미지에서 두 실행 결과를 비교할 때 유용합니다.

9.6 식생 분리도 평가

NDVI를 독립적인 Ground Truth로 사용해 알고리즘이 식생 픽셀을 얼마나 정확하게 찾아냈는지 평가합니다. 즉, "NDVI가 식생이라고 말하는 픽셀 중 몇 %가 잎 클래스에 들어갔는가?"를 답합니다.

지표정의해석
Recall (검출률) 실제 양성 검출률 NDVI > 0.15인 픽셀 중 잎 클래스에 들어간 비율.
낮으면: 잎 픽셀이 토양/배경에 섞임 → NDVI 임계값을 낮추세요.
Precision (정밀도) 양성 예측 정확도 잎 클래스에 속한 픽셀 중 실제로 NDVI > 0.15인 비율.
낮으면: 비식생이 잎으로 잘못 분류됨 → NDVI 임계값을 높이세요.
F1 점수 Recall과 Precision의 조화평균 단일 종합 점수. 0.85 이상이면 우수. 0.60 미만이면 식생/비식생 분류에 심각한 오류가 있음.
ℹ
참고 이 지표는 이미지에 NIR 및 Red 밴드가 있어 NDVI 계산이 가능할 때만 표시됩니다. "식생 분리도 데이터가 없습니다"가 표시되면 해당 밴드가 없는 이미지일 수 있습니다.
