User Manual

Hyperspectral Field Crop
Analysis Tool

Unsupervised Edition — No Labels Required

This tool automatically classifies pixels in hyperspectral field crop images and extracts per-class reflectance spectra. It runs entirely on your local machine — no cloud, no internet, no labelled training data required. Results are saved as a CSV file and a fully interactive HTML report.

🌱 Hybrid (NDVI + K-means)
📊 K-Means Clustering
📐 Spectral Angle Mapping
🤖 Autoencoder (Deep Learning)
📄 Interactive HTML Report
📈 Per-Class Spectra CSV
GitHub: https://github.com/hoonsoolee/Extract_spectra   |   Python 3.9+  |  Runs on CPU  |  Windows / macOS / Linux

Installation

One-time setup — takes about 5 minutes

Requirements

RequirementNotes
Python 3.9 or laterDownload from python.org. During installation on Windows, check "Add Python to PATH".
GitDownload from git-scm.com. Used to download the tool from GitHub.
8 GB RAMRecommended minimum. Larger images (2000×2000 px+) may need 16 GB.
PyTorch (optional)Required only for the Autoencoder method. See Step 3 below.

Installation Steps

  1. Open a terminal (Windows: press Win+R, type cmd; Mac/Linux: open Terminal).

  2. Download the tool from GitHub:

    git clone https://github.com/hoonsoolee/Extract_spectra.git
cd Extract_spectra

  3. Install all required Python packages at once:

    pip install -r requirements.txt

  4. (Optional) Install PyTorch — only needed for the Autoencoder method:

    pip install torch

  5. Verify the installation by starting the app:

    python -m streamlit run app_unsupervised.py

    Your browser should open automatically at http://localhost:8501.

Placing Your Data

Create a folder anywhere on your computer (e.g. C:/data/hyperspectral or ./data) and copy your image files into it. Subfolders are supported — the tool scans recursively.

For ENVI files: make sure the .hdr file and its data file (.raw, .bil, etc.) share the same base filename and are in the same folder.

  5. Results saved to output/<filename>/

Best for: processing an entire dataset; each file gets its own report.

  2. Click Run Analysis.

Estimating Batch Time Run Single File mode on one image first. The elapsed time shown in the success banner is a reliable estimate of per-file processing time. Total batch time ≈ single-file time × number of files.

Classification Methods

How the algorithm groups pixels into classes — no labels needed

All four methods are fully unsupervised — they discover natural spectral groupings automatically. Choose based on your image type and analysis goal.

Recommended

Hybrid (NDVI + Brightness + K-means)

Uses vegetation physics first, then statistical clustering. Best for standard field crop images with NIR coverage.

General Purpose

K-Means

Pure data-driven clustering. Works on any image regardless of spectral range. No vegetation index required.

Lighting Invariant

SAM — Spectral Angle Mapping

Compares the shape of spectra, not their brightness. Best for images with strong shadows or uneven lighting.

Deep Learning

Autoencoder

Neural network learns compressed spectral features, then clusters them. Requires PyTorch. Slowest but most flexible.

5.1 Hybrid — Recommended Default

The Hybrid method combines three techniques in sequence. It is the recommended starting point because it leverages well-established vegetation physics rather than relying purely on statistical grouping.

1
NDVI Mask
NDVI = (NIR − Red) / (NIR + Red)
Pixels above threshold → Vegetation
Others → Non-vegetation
→
2
Brightness Mask
Within vegetation pixels:
Low mean reflectance → Shadow
High reflectance → Sunlit
→
3
K-Means Refinement
Within each segment, K-means finds finer sub-classes (e.g. different crop varieties or soil types)

Parameters

ParameterDefaultRangeWhat it controls
NDVI Threshold 0.15 0.05 – 0.30 Pixels with NDVI ≥ this value are classified as vegetation (leaves).
↑ Increase if non-plant pixels are incorrectly included in leaf classes.
↓ Decrease if too few leaf pixels are being detected (stressed/senescent crops).
Brightness Threshold 0.08 0.03 – 0.20 Within vegetation pixels, those with mean reflectance below this value are classified as shadow.
↑ Increase in strongly shaded images.
↓ Decrease in bright or overcast conditions.

Output Class IDs

IDClass NameDescription
0BackgroundNon-vegetation, non-soil (paths, edges, sky)
1Sunlit LeavesVegetation pixels in direct sunlight — high reflectance
2Shadowed LeavesVegetation pixels in shadow — low reflectance
3SoilBare soil and ground cover
4OtherUnclassified pixels
💡
Tuning Workflow Run once with default settings. Open the HTML report and look at the Per-Class Images section. If a class clearly contains mixed pixel types, adjust the relevant threshold and re-run. Usually 1–2 adjustments are enough.
⚠
No NIR Band? NDVI requires a Near-Infrared (~800 nm) and Red (~660 nm) band. If your image does not cover these wavelengths, the Hybrid method automatically falls back to K-Means. Switch to K-Means or SAM manually in that case.

5.2 K-Means

A classic unsupervised clustering algorithm. Makes no assumptions about vegetation indices or spectral bands — it works purely from the statistical structure of the data.

How it works

  1. PCA reduces the hundreds of spectral bands down to 15 principal components, capturing most of the variance while discarding noise.

  2. K-Means partitions the compressed spectral space into K clusters by iteratively minimising the distance from each pixel spectrum to its cluster centroid.

  3. Each cluster becomes one output class. The classes are colour-coded but not named — you interpret them from the spectral chart in the report.

Parameter

ParameterDefaultWhat it controls
Number of Classes 6 How many clusters to find. More clusters = finer separation but harder to interpret. Start with 6 and adjust after reviewing the report.
💡
When to use K-Means Use K-Means when your image lacks NIR bands (making NDVI unavailable), when you want a purely data-driven result without any vegetation-index assumptions, or as a quick comparison baseline against Hybrid results.

5.3 SAM — Spectral Angle Mapping

SAM compares spectra by measuring the angle between spectral vectors rather than their Euclidean distance. This makes it insensitive to illumination intensity — a sunlit and a shadowed pixel of the same material will have a small angle, even though their absolute reflectance values differ significantly.

How it works

  1. Endmember extraction: Representative spectra are automatically extracted from the image using PCA — no user-provided reference spectra needed.

  2. Angle comparison: Each pixel's spectrum is compared to every endmember. The spectral angle (in radians) is calculated — small angle = high similarity.

  3. Assignment: Each pixel is assigned to the class of the endmember with the smallest angle.

  4. Background threshold: If the smallest angle exceeds the angle threshold, the pixel is assigned to Background (class 0) — it does not sufficiently resemble any known class.

Parameters

ParameterDefaultWhat it controls
Angle Threshold 0.10 rad (~5.7°) Maximum allowed angle between a pixel and its nearest endmember. Pixels exceeding this are assigned to Background.
↑ Increase if too many pixels are Background (threshold too strict).
↓ Decrease for stricter matching.
Number of Classes 6 Number of endmembers (representative spectra) to extract and use as class prototypes.
💡
When to use SAM SAM is particularly useful for images with strong shadow gradients or varying illumination — such as images taken near sunrise/sunset, under partially cloudy conditions, or with topographic shading.

5.4 Autoencoder (Deep Learning)

The Autoencoder uses a small neural network to learn a compressed representation of each pixel's spectrum. K-Means clustering is then applied in this learned space, which can capture non-linear spectral relationships that PCA (used by the K-Means method) might miss.

How it works

  1. Encoder: An MLP compresses each spectrum (hundreds of bands) down to 16 latent dimensions.

  2. Decoder: Another MLP reconstructs the original spectrum from the 16-dimensional code. The network trains by minimising reconstruction error.

  3. Clustering: After training, K-Means clusters the 16-dim latent vectors to produce the final class assignments.

Parameters

ParameterDefaultWhat it controls
Training Epochs 60 Number of passes over the training data. More epochs = better compression, but longer runtime. Reduce to 20–30 for a quick test; increase to 100–150 for very complex images.
Number of Classes 6 Number of K-Means clusters applied to the latent space.
⚠
PyTorch Required The Autoencoder method requires PyTorch. Install it with: pip install torch
If PyTorch is not installed, selecting Autoencoder will cause an error when you click Run.

Number of Classes

How many groups to divide the image into

The slider controls how many output groups (clusters) the algorithm will create. The right value depends on the complexity of your scene.

ValueWhen to use
2 – 3Very simple scenes: one crop type, uniform soil, minimal background variation.
4 – 6Typical field images. Recommended starting range.
7 – 10Complex scenes: multiple crop varieties, varied soil moisture, deep shadow patches.
11+Use only if you have a specific scientific reason. Results become hard to interpret.
💡
How to pick the right number Start with 6. Open the HTML report and look at the Per-Class Images. If two classes look visually identical (same spatial distribution, similar spectra), reduce the count. If one class clearly contains a mix of different materials, increase it.

Running the Analysis

Step-by-step from configuration to results

  1. Configure all settings in the sidebar: Data Source, Processing Mode, Method, Parameters, and Output Folder.

  2. Click the blue "🚀 Run Analysis" button at the bottom of the sidebar.

  3. A spinner shows: "Analysing… (may take several minutes for large images)". Do not close the browser tab while processing.

  4. When done, a green banner appears: ✅ Analysis complete! · ⏱ Xs. The elapsed time helps you estimate full batch duration.

  5. The HTML report path is displayed — click or copy it to open in your browser.

  6. Classification map thumbnails appear as a quick preview.

  7. The run log at the bottom of the page shows per-step timing and quality metrics.

Understanding the HTML Report

A complete breakdown of every section in the output report

After each run, a file named report_YYYYMMDD_HHMMSS.html is saved in the output folder. Open it in any browser (Chrome, Firefox, Edge, Safari). It is fully self-contained — no internet required.

9.1 Image Overview

The first card displays three side-by-side images:

ImageBands usedWhat to look for
RGB Composite Red ~660 nm, Green ~550 nm, Blue ~450 nm Natural-colour view of the field. Confirms the data loaded correctly and the scene looks as expected.
CIR False-Color NIR ~800 nm → Red, Red → Green, Green → Blue Healthy vegetation appears bright red/magenta. Use this to visually verify where vegetation is — it should align with the leaf classes in the classification map.
Classification Map All bands (output of algorithm) All classes shown at once in their assigned colours with a legend showing pixel count and percentage. First overall check of the classification result.

9.2 Per-Class Classification Images

One large image per class. The pixels belonging to that class are shown in the class colour; all other pixels are shown as a darkened greyscale background. This makes it easy to visually inspect each class in isolation.

Good classification:

The class pixels form a spatially coherent region — e.g. the entire leaf canopy, a contiguous patch of soil, or the shadowed side of rows.

Needs adjustment:

The class pixels are scattered randomly, or visually mix different materials (e.g. soil and leaves in the same class). Adjust parameters and re-run.

9.3 Classification Summary Table

A table listing each class with its pixel count, percentage of total image area, and a horizontal colour bar showing the proportion visually.

Check whether the distribution looks reasonable for your scene: a dense canopy image should have large leaf classes; a sparse crop image will have more soil/background.

9.4 Reflectance Spectra Chart

An interactive Plotly chart showing the mean reflectance spectrum for each class, plotted against wavelength (nm) or band index. The shaded band around each line is ±1 standard deviation, indicating spectral variability within the class.

Reset zoomDouble-click on the chart

💡
Interpreting the spectra Well-separated class spectra (clear vertical distance between lines across most wavelengths) indicate that the classes are spectrally distinct — a strong sign of good classification. If two classes have nearly identical spectra, consider merging them by reducing the number of classes.

9.5 Quality Assessment

Two metrics are automatically calculated to quantify how well the clusters are separated in spectral space.

Cluster Quality Score (Silhouette-based gauge)

The Silhouette Score (range: −1 to +1) measures how similar each pixel's spectrum is to its own class versus the nearest other class. It is converted to a 0–100% gauge for readability.

0% — Poor 25% 50% — Fair 75% — Good 100% — Excellent
ScoreRatingMeaning
90 – 100%ExcellentClasses are very well separated in spectral space.
75 – 90%GoodReliable classification for most scientific purposes.
50 – 75%FairSome class overlap. Consider adjusting parameters or trying a different method.
Below 50%PoorSignificant overlap. Try a different method, change the number of classes, or adjust thresholds.

Davies-Bouldin Index

Measures the ratio of within-cluster scatter to between-cluster separation. Lower is better. Values below 1.0 are generally good; values above 2.0 suggest significant class overlap. Unlike the Silhouette Score, there is no universal "ideal" cutoff — it is most useful for comparing two runs of the same image.

9.6 Vegetation Separation Assessment

This section evaluates how accurately the algorithm identified vegetation pixels, using NDVI as an independent ground truth. It answers: of all pixels that NDVI says are vegetation, how many were correctly labelled as leaves?

MetricDefinitionHow to interpret
Recall True positive rate Of all pixels with NDVI > 0.15, what fraction ended up in a leaf class?
Low recall: leaf pixels are being missed, placed in soil/background. → Lower the NDVI threshold.
Precision Positive predictive value Of all pixels placed in a leaf class, what fraction truly have NDVI > 0.15?
Low precision: non-vegetation is being labelled as leaves. → Raise the NDVI threshold.
F1 Score Harmonic mean of Recall & Precision A single combined score. Above 0.85 is excellent. Below 0.60 indicates significant misclassification of vegetation vs. non-vegetation.
ℹ
Note This metric is only shown for the Hybrid and K-Means methods, and only when the image contains NIR and Red bands for NDVI computation. If the section shows "No vegetation separation data available", your image may lack the required bands.

Troubleshooting

Common issues and how to fix them
ProblemCauseSolution
ModuleNotFoundError A required Python package is not installed. Run: pip install -r requirements.txt
For Autoencoder: pip install torch
Unsupported format error The file extension is not recognised. Check the extension is .hdr, .tif, .tiff, .h5, .hdf5, or .mat.
ENVI file does not load The .hdr and data file are in different folders or have different names. Make sure the .hdr and its data file (.raw / .bil etc.) share the same base name and are in the same folder.
Classification map is mostly one colour NDVI threshold is too high, or the wrong number of classes. Lower the NDVI threshold (Hybrid) or try a different number of classes. Check the Quality Assessment score.
Too many Background pixels (SAM) Angle threshold is too strict. Increase the Angle Threshold (e.g. 0.10 → 0.20).
Processing is very slow Large image size or many epochs (Autoencoder). Set spatial_downsample: 2 in config.yaml to halve image dimensions. Reduce Autoencoder epochs to 20–30 for testing.
Browser does not open automatically Browser launch failed silently. Open http://localhost:8501 manually.
Port 8501 already in use Another Streamlit app is running on the same port. Run: python -m streamlit run app_unsupervised.py --server.port 8502

Useful Tips

Testing a new dataset

  • Set File Limit = 1
  • Use Single File mode
  • Run Hybrid with default settings
  • Check Per-Class Images first
  • Then adjust one parameter at a time

Improving poor results

  • Check Quality Score in report
  • Look at Per-Class Images
  • Try a different method
  • Adjust number of classes (+/-2)
  • Tune NDVI or Angle threshold

Using the spectra.csv

  • Python: pd.read_csv("spectra.csv")
  • R: read.csv("spectra.csv")
  • Excel: File → Open directly
  • Rows = spectral bands
  • Columns = class mean / std

Saving the report as PDF

  • Open the .html in Chrome
  • Press Ctrl + P
  • Destination: "Save as PDF"
  • Disable headers/footers
  • Click Save
